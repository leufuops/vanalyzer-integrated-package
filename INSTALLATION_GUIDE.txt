OVERVIEW
================================================================================

vAnalyzer is an enterprise vulnerability reporting platform that integrates
with Vicarius vRx to provide risk-based vulnerability assessment. The platform
enriches vulnerability data with threat intelligence from CISA KEV, FIRST EPSS,
and optional advanced threat intelligence sources.

Technology Stack: Docker Swarm, PostgreSQL, Metabase, Traefik, Python

================================================================================
SYSTEM REQUIREMENTS
================================================================================

Hardware
- CPU: 4+ cores (8+ recommended)
- RAM: 8GB minimum (16GB recommended)
- Storage: 50GB minimum (SSD preferred)

Software
- OS: Linux (Ubuntu 20.04+, RHEL/Rocky/AlmaLinux 8+)
- Docker: 20.10+ (installer provided)
- Access: Root or sudo privileges

Required Credentials
- Vicarius Dashboard ID (e.g., company.vicarius.cloud or company)
- Vicarius API Key (from Settings → API Keys in Vicarius dashboard)

================================================================================
INSTALLATION
================================================================================

Step 1: Install Docker
    sudo ./vanalyzer install

Step 2: Configure System
    sudo ./vanalyzer init

Configuration Parameters:
  - Hostname: FQDN for SSL (e.g., vanalyzer.company.com)
  - Dashboard ID: Vicarius instance identifier
  - API Key: Vicarius authentication key
  - Database Credentials: Username, database name, password
  - Sync Interval: Hours between data refresh (default: 6)
  - External Data: Enable CISA KEV/EPSS integration (recommended)
  - Threat Intelligence: Enable advanced threat intelligence API (optional)

Step 3: Deploy Stack
    sudo ./vanalyzer deploy

Deployment Time: 5-10 minutes
Initial Data Sync: 15-30 minutes

Step 4: Configure DNS
Add hostname to your hosts file:
  - Linux/Mac: /etc/hosts
  - Windows: C:\Windows\System32\drivers\etc\hosts

    SERVER_IP    vanalyzer.company.com

Step 5: Access Dashboard
1. Navigate to: https://[your-hostname]
2. Accept self-signed certificate warning
3. Login with default credentials:
   - Username: vrxadmin@vrxadmin.com
   - Password: Vicarius123!@#
4. Change password immediately (Settings → Account)
5. Configure database connection (Settings → Admin Settings → Databases)
6. Wait for initial sync to complete (~15-30 minutes)

================================================================================
DAILY OPERATIONS
================================================================================

Monitoring Commands
    sudo ./vanalyzer status      # Check service status
    sudo ./vanalyzer health      # Run comprehensive health checks
    sudo ./vanalyzer validate    # Verify configuration

Log Management
    sudo ./vanalyzer logs app        # Application sync logs
    sudo ./vanalyzer logs appdb      # Database logs
    sudo ./vanalyzer logs metabase   # Dashboard logs
    sudo ./vanalyzer logs traefik    # Proxy/SSL logs

Data Management
    sudo ./vanalyzer update                      # Force immediate data sync
    sudo ./vanalyzer backup                      # Create timestamped backup
    sudo ./vanalyzer restore <backup_dir>        # Restore full system
    sudo ./vanalyzer restore-metabase <backup>   # Restore dashboard only

Data Synchronization
- Automatic: Runs at configured interval (default: every 6 hours)
- Manual: Execute sudo ./vanalyzer update
- Monitor: sudo ./vanalyzer logs app | grep -i sync

================================================================================
OFFLINE/AIR-GAP DEPLOYMENT
================================================================================

Create Offline Bundle (Internet-Connected System)
    sudo ./vanalyzer deploy
    sudo ./vanalyzer bundle

Deploy in Air-Gapped Environment
    sudo ./vanalyzer import
    sudo ./vanalyzer init
    sudo ./vanalyzer deploy

Note: Offline deployments will not receive external threat intelligence
updates (KEV/EPSS).

================================================================================
TROUBLESHOOTING
================================================================================

Quick Diagnostics
    sudo ./vanalyzer health      # Comprehensive system check
    sudo ./vanalyzer validate    # Configuration validation
    sudo ./vanalyzer status      # Service status overview

Common Issues

No Data in Dashboards
    # Check sync logs
    sudo ./vanalyzer logs app | grep -i sync

    # Force manual sync
    sudo ./vanalyzer update

    # Wait 15-30 minutes for initial data population

Cannot Access Dashboard
    - Verify services: sudo ./vanalyzer status
    - Check hostname is correctly added to hosts file
    - Ensure you're accessing via FQDN (e.g., https://vanalyzer.company.com)
    - Note: Direct IP access is not supported; FQDN is required due to
      Traefik routing configuration

Certificate Warnings
    - Normal for self-signed certificates
    - Click "Advanced" → "Proceed"
    - For production, replace with valid certificates in traefik/config/certs/

Database Connection Issues
    - Verify credentials match configuration
    - Check appdb service: sudo ./vanalyzer logs appdb
    - Restart services: sudo ./vanalyzer deploy

ARCHITECTURE OVERVIEW
================================================================================

Service Components
- app - Python data sync engine with scheduled jobs
- appdb - PostgreSQL database for vulnerability data
- metabase - Web analytics dashboard
- traefik - Reverse proxy with SSL termination

Data Flow
1. Initial Sync: Fetches Vicarius vRx data and external threat intelligence
2. Scheduled Sync: Runs every N hours (configurable)
   - Refreshes base vulnerability tables
   - Syncs active vulnerabilities
   - Updates differential tracking tables
   - Enriches with KEV/EPSS/threat intelligence data

Key Features
- Threat Intelligence Integration: CISA KEV, FIRST EPSS, optional advanced
  sources
- CVRA Scoring: Composite Vulnerability Risk Assessment (0-100 scale)
- Automated Enrichment: Multi-source threat context
- Analytics Dashboard: Metabase-powered visualizations
